# BT-004 Design Decisions

This file records design decisions, rejected ideas, pivots, and tradeoffs during BT-004 and will be removed at the end of the work.

## 2026-03-23

- Decision:
  - treat BT-004 as a nuclear redesign rather than a cosmetic polish pass, and treat Fase 1 as the point where Module 1 and Module 2 become true implementation work
- Why:
  - the brief intentionally calls for a stronger visual system, clearer hierarchy, and a more deliberate premium feel
- Alternatives rejected:
  - limiting the work to "just" polish
  - keeping the existing fragmented panel proportions
- Follow-up:
  - continue the redesign in later phases with the teleprompter, command palette, diagram focus, and issues surfaces

- Decision:
  - separate the BT-004 notes into brief, implemented, and decisions files
- Why:
  - the design intent, delivery record, and pivots stay easier to evolve independently
- Alternatives rejected:
  - keeping one monolithic BT-004 markdown file
- Follow-up:
  - remove the three files at the end of BT-004 if they no longer need to live in the repository

- Decision:
  - use `--font-sans`, `--font-mono`, `--bg-base`, and `--bg-surface` as the canonical vocabulary for the new visual system
- Why:
  - the new naming makes the design brief map directly to the implemented shell and reduces confusion between old and new token names
- Alternatives rejected:
  - keeping `--font-family-*` as the primary public vocabulary
  - leaving the old surface names as the only contract
- Follow-up:
  - later phases should continue to prefer the new names and only keep aliases where compatibility needs it
