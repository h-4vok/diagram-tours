# Vision Log

## Vision
- Diagram-native documentation.
- Diagram + tour + interactive player.
- Target use cases: architecture walkthroughs, onboarding, runbooks, teaching.
- Goal: executable explanations, not static diagrams.

## Historical Framing
- Original concept: guided walkthroughs attached directly to diagram nodes.
- Diagram stays primary artifact; steps add narrative.

## Non-Current / Future UX Direction
- More polished minimap/navigation patterns.
- Richer timeline models.
- Broader node-click navigation patterns.
- Explorer/tree navigation based on source structure.
- Faster author preview flows.
- Richer author diagnostics.
- Advanced viewport behavior: zoom-to-fit, animation, smarter group centering, constraints.

## Philosophy
- Diagram-first.
- Minimal author friction.
- Visual clarity.
- Strong developer tooling.

## Known/Evolving Areas
- Viewport edge cases on very large or distant-focus diagrams.
- Highlight hierarchy refinement for labels/connectors.
- Ongoing layout polish across overlays and diagram canvas.

## Prior Focus / Status Notes
- Earlier focus: viewport centering, multi-focus, highlight hierarchy, canvas-first layout, overlay architecture.
- Earlier status: parser + player stable; current work shifts toward UX/navigation refinement.
