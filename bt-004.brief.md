# 1. Fundamentos Visuales (Design Tokens & Theming)

Este módulo define las reglas estéticas base para `diagram-tours`. El objetivo es lograr una interfaz de grado "herramienta de ingeniería premium" (inspirada en Vercel, Linear, Raycast), alejándonos de estilos genéricos o de plantillas. Se prioriza el contraste técnico, la legibilidad prolongada y el uso de bordes/desenfoques sobre sombras expansivas.

## 1.1. Tipografía

Debemos reemplazar las fuentes del sistema o genéricas por un dúo tipográfico de alta legibilidad en pantallas densas.

* **Fuente Principal (UI y Lectura Larga):** `Geist` o `Inter`.
    * Uso: Headers, menús, descripciones de pasos ("teleprompter"), botones.
    * Variable CSS: `--font-sans: 'Geist', 'Inter', sans-serif;`
* **Fuente Técnica (Nodos, Código, Referencias):** `Geist Mono` o `JetBrains Mono`.
    * Uso: Etiquetas dentro del diagrama Mermaid, nombres de archivos, atajos de teclado (`Cmd+K`), términos técnicos resaltados en las explicaciones.
    * Variable CSS: `--font-mono: 'Geist Mono', 'JetBrains Mono', monospace;`

*Regla de estilo:* Eliminar el uso de `text-transform: uppercase` (ALL CAPS) en etiquetas de UI (como "NAVIGATION MINIMAP"). Usar *Title Case* o *Sentence Case* con `font-weight: 500` o `600`.

## 1.2. Paleta de Colores (Theming)

Huir de los negros puros y blancos nucleares absolutos para reducir la fatiga visual.

**Dark Theme (Modo Grafito/Pizarra):**
* **Background Base (Canvas):** `--bg-base: #0E1116;` (Un gris azulado muy oscuro, no negro).
* **Background Surface (Paneles/Modales):** `--bg-surface: #161B22;`
* **Texto Principal:** `--text-primary: #E6EDF3;` (Evita el #FFFFFF puro).
* **Texto Secundario (Mutado):** `--text-secondary: #8B949E;`
* **Bordes (Crucial para separar paneles):** `--border-subtle: rgba(255, 255, 255, 0.1);`
* **Acento (Focus/Highlights):** `--accent-primary: #3B82F6;` (Azul eléctrico técnico) o `--accent-warning: #F59E0B;` (Ámbar para alertas).

**Light Theme (Modo Nítido):**
* **Background Base (Canvas):** `--bg-base: #FAFAFA;`
* **Background Surface (Paneles/Modales):** `--bg-surface: #FFFFFF;`
* **Texto Principal:** `--text-primary: #111827;`
* **Texto Secundario (Mutado):** `--text-secondary: #6B7280;`
* **Bordes:** `--border-subtle: rgba(0, 0, 0, 0.08);`
* **Acento (Focus/Highlights):** Igual al Dark Theme, ajustando opacidad si es necesario.

## 1.3. Elevación, Profundidad y Efectos

El estilo premium actual minimiza las sombras proyectadas en modos oscuros y confía en el contraste de bordes y el *Glassmorphism*.

* **Elevación Flotante (Paneles, Command Palette, Teleprompter):**
    * *No usar:* `box-shadow: 0 10px 15px rgba(0,0,0,0.5);` (ensucia el diagrama).
    * *Usar:* Un borde sólido de 1px usando `--border-subtle` y una sombra hiper-dura e imperceptible.
    * *CSS:* `border: 1px solid var(--border-subtle); box-shadow: 0 4px 24px rgba(0,0,0,0.4);` (en dark mode).
* **Glassmorphism (El efecto "Subtítulo"):**
    * Para el panel inferior de lectura de pasos, necesitamos que el diagrama de fondo se intente adivinar sin estorbar la lectura.
    * *Efecto:* `backdrop-filter: blur(12px) saturate(150%);` combinado con un fondo semitransparente (`rgba(22, 27, 34, 0.8)` en Dark Mode).
* **Radios de Borde (Border Radius):**
    * Mantener la UI "crujiente" pero amigable. Usar `--radius-md: 8px;` para botones y modales medianos, y `--radius-lg: 12px;` para contenedores grandes (Command Palette, Teleprompter).

# 2. Arquitectura de Pantalla y Layout Global

El objetivo de este módulo es maximizar el área visible del diagrama (*canvas*) y eliminar la fragmentación visual. Pasamos de un modelo de "islas flotantes dispersas" a un modelo de "lienzo inmersivo con controles perimetrales anclados".

## 2.1. El Contenedor Principal (The Shell)

La aplicación debe comportarse como una aplicación de escritorio a pantalla completa, sin *scroll* general en la ventana.

* **Layout Base:** El contenedor raíz (`#app` o `main`) debe tener `width: 100vw`, `height: 100vh` y `overflow: hidden`.
* **El Canvas:** El contenedor del diagrama Mermaid ocupará el 100% de este espacio y se ubicará en la capa inferior (`z-index: 0`). Todo el resto de la interfaz flotará sobre él.

## 2.2. Top Bar (Header)

El *header* actual ocupa todo el ancho y tiene un fondo sólido que corta la pantalla. Vamos a hacerlo más ligero y funcional.

* **Estructura:** Flexbox horizontal (`justify-content: space-between`, `align-items: center`).
* **Fondo:** Translúcido con desenfoque (`backdrop-filter: blur(8px)`) y borde inferior de 1px (`border-bottom: 1px solid var(--border-subtle)`).
* **Distribución de elementos:**
    * **Izquierda:** Logo/Nombre minimalista + Breadcrumbs (Ej. `diagram-tours / flowchart / Payment Flow`). Los breadcrumbs usan `--font-mono` en tamaño pequeño.
    * **Centro:** Un botón sutil que funcione como pista (*hint*) para el buscador: `[🔍 Search tours... ⌘K]`.
    * **Derecha:** Controles globales (Toggle de Light/Dark mode, enlace a GitHub/Docs).

## 2.3. Reubicación de Controles Auxiliares (Zoom y Minimapa)

En lugar de tener el Minimapa por un lado y el Zoom por otro (como en las capturas originales), vamos a consolidarlos en un único "Cluster de Control" para reducir el ruido visual.

* **Posición:** Esquina inferior derecha (`bottom: 24px`, `right: 24px`), con `position: absolute`.
* **Layout del Cluster:**
    * Un contenedor vertical que agrupa ambos elementos.
    * Arriba: El Minimapa (compacto, sin el título enorme de "NAVIGATION MINIMAP").
    * Abajo: Controles de zoom (`+`, `-`, `Fit`) integrados como una botonera segmentada (tipo *pill* o cápsula), pegada al minimapa para que se perciban como una sola unidad de control de cámara.

## 2.4. Gestión de Capas (Stacking Context / Z-Index)

Para evitar que los nodos del diagrama se superpongan accidentalmente con la UI, estandarizamos las capas:

* `z-index: 0` -> Canvas interactivo de Mermaid (fondo).
* `z-index: 10` -> Top Bar (Header) y Cluster de Control (Zoom/Minimapa).
* `z-index: 20` -> Teleprompter (Navegador de pasos, centrado abajo).
* `z-index: 50` -> Command Palette (Modal de búsqueda).
* `z-index: 100` -> Toasts y Alertas críticas.

# 3. El "Teleprompter" (Navegación de Pasos)

Este componente reemplaza las tarjetas laterales tradicionales. Se conceptualiza como los "subtítulos" de un reproductor multimedia. Su objetivo es permitir la lectura fluida de la explicación paso a paso sin obligar al usuario a alejar la vista del diagrama central.

## 3.1. Posicionamiento y Dimensiones

El contenedor debe flotar en la parte inferior central de la pantalla, anclado de forma absoluta sobre el *canvas*.

* **Posición:** `position: absolute; bottom: 32px; left: 50%; transform: translateX(-50%);`
* **Ancho (Width):** Dinámico pero restringido. `width: 100%; max-width: 768px;` (El límite de 768px garantiza que las líneas de texto no superen los 70-80 caracteres, que es el estándar ergonómico óptimo para lectura).
* **Padding:** Generoso para respirar. `padding: 24px 32px;`
* **Z-Index:** `z-index: 20;` (Por encima del canvas y del cluster de zoom).

## 3.2. Estilo Visual (Glassmorphism Premium)

Dado que este panel puede superponerse visualmente a partes del diagrama que quedan por debajo, debe usar un efecto de cristal esmerilado para mantener el contexto espacial sin sacrificar la legibilidad del texto.

* **Fondo (Dark Mode):** `background: rgba(22, 27, 34, 0.75);` (Un gris grafito al 75% de opacidad).
* **Filtro (Blur):** `backdrop-filter: blur(16px) saturate(120%);`
* **Borde:** `border: 1px solid var(--border-subtle);`
* **Radio:** `border-radius: var(--radius-lg);` (Ej. 16px).
* **Sombra:** `box-shadow: 0 20px 40px -8px rgba(0, 0, 0, 0.5);` (Para "despegarlo" del diagrama de fondo).

## 3.3. Anatomía del Contenido y Tipografía

El panel se divide en tres áreas horizontales: Controles Izquierdos, Texto Central, Controles Derechos.

1. **Indicador de Progreso (Tope superior):**
   * Una barra de progreso ultra fina (`height: 2px`) en el borde superior interno del panel, usando `--accent-primary`. Alternativamente, un texto muy sutil en la esquina superior izquierda: `Step 2 of 5` usando `--font-mono` en `font-size: 0.75rem` y `--text-secondary`.

2. **Texto Explicativo (El "Subtítulo"):**
   * **Tipografía:** `--font-sans`, `font-size: 1.125rem` (18px) o `1.25rem` (20px) para máxima legibilidad.
   * **Interlineado:** `line-height: 1.6;`
   * **Color:** `--text-primary`.
   * **Formateo Inline:** Cualquier referencia a un nodo del diagrama (ej. `API Gateway`) debe renderizarse con un tag `<code>` que use `--font-mono`, tamaño reducido (`0.9em`), color `--accent-primary` y un pequeño fondo semitransparente.

3. **Controles de Navegación:**
   * **Botones Prev/Next:** Ocultos o muy sutiles en los flancos. Idealmente botones circulares de tamaño mediano (`40x40px`) que solo aparezcan o se resalten al pasar el cursor (*hover*).
   * **Atajos Visibles:** Al lado de los botones o en una esquina, mostrar la pista del atajo de teclado usando el estilo `<kbd>`: `[ ← ]` para atrás, `[ Espacio ]` o `[ → ]` para avanzar.

## 3.4. Transiciones y Animaciones

El cambio de un paso a otro no debe ser un corte abrupto. 

* **Transición de Texto:** Al hacer "Next", el texto actual hace un `fade-out` rápido (100ms) y el texto nuevo hace un `fade-in` (200ms) con un ligerísimo desplazamiento desde abajo (`translateY: 4px` a `0`).
* **Resize Automático:** Si el nuevo paso tiene más o menos texto, la altura del contenedor principal debe animarse suavemente (`transition: height 0.3s ease-in-out`) para evitar saltos bruscos en la interfaz.

# 4. La Command Palette (Buscador y Navegación de Tours)

Este módulo reemplaza el antiguo menú desplegable (dropdown) de la barra lateral por un modal de búsqueda centrado y priorizado para el uso del teclado (*keyboard-first*). Inspirado en Raycast, Spotlight y Vercel.

## 4.1. Comportamiento y Disparadores (Triggers)

La Command Palette debe sentirse instantánea.

* **Atajo Principal:** `Cmd + K` (Mac) o `Ctrl + K` (Windows/Linux).
* **Atajo Secundario:** Presionar la tecla `/` (muy común en herramientas técnicas).
* **UI Trigger:** Un botón en el *Header* (como definimos en el Módulo 2) que muestre el ícono de lupa y el atajo `[ ⌘K ]`.
* **Cierre:** Tecla `Esc` o haciendo clic fuera del modal (en el *backdrop*).

## 4.2. Estructura Visual del Modal

El modal debe aparecer en el centro superior de la pantalla, atrayendo la atención sin tapar todo el contexto subyacente.

* **Backdrop (Fondo oscuro):** Al abrirse, el canvas se oscurece. `background: rgba(0, 0, 0, 0.4); backdrop-filter: blur(4px); z-index: 50;`
* **Contenedor Principal:**
    * Posición: Centrado horizontalmente, con un margen superior. `margin-top: 12vh; width: 100%; max-width: 640px;`
    * Fondo: `--bg-surface` (ej. `#161B22` en dark mode).
    * Borde: `1px solid var(--border-subtle)`.
    * Radio: `border-radius: var(--radius-lg);` (ej. 12px o 16px).
    * Sombra: Elevación máxima. `box-shadow: 0 24px 48px -12px rgba(0, 0, 0, 0.7);`
    * *Overflow*: `overflow: hidden;` (Para que el input y la lista respeten los bordes redondeados).

## 4.3. El Input de Búsqueda (Header del Modal)

El área de texto debe ser prominente y no parecer un formulario web tradicional.

* **Altura:** Generosa (ej. `64px`).
* **Tipografía:** `--font-sans`, grande (`font-size: 1.25rem` o 20px), color `--text-primary`.
* **Estilo:** Sin bordes visibles (`border: none`, `outline: none`), fondo transparente.
* **Placeholder:** "Search tours, diagrams, or folders..." en color `--text-secondary`.
* **Separador:** Un borde inferior sutil (`border-bottom: 1px solid var(--border-subtle)`) que lo separe de los resultados.

## 4.4. Lista de Resultados y Metadatos

La lista debe ser escaneable. En lugar de mostrar solo el "path" crudo, vamos a estructurar la información.

* **Agrupación (Categories):** Los resultados deben dividirse en secciones discretas: `Favorites` (con un ícono de estrella), `Recent`, y `All Diagrams`. Los títulos de sección usan `--font-sans`, `font-size: 0.75rem`, en mayúsculas sostenidas muy sutiles (`letter-spacing: 0.05em`) y color `--text-secondary`.
* **Anatomía del Item (Fila):**
    * **Alto:** `height: 48px;` con `display: flex; align-items: center; padding: 0 16px;`
    * **Ícono Izquierdo:** Un ícono SVG que indique si es *Flowchart* o *Sequence Diagram*.
    * **Nombre:** El nombre legible del tour en `--text-primary`.
    * **Path (Ruta):** A la derecha del nombre o debajo, la ruta del archivo en tamaño pequeño (`0.8rem`), color `--text-secondary`, y usando `--font-mono` (ej. `fixtures/flowchart/payment-flow.yaml`).
    * **Metadatos (Derecha):** Una pequeña píldora (*badge*) que diga, por ejemplo, `4 steps`.
* **Interacción y Estados (Estados de Foco):**
    * Al navegar con las flechas del teclado (`↑` / `↓`), el item activo cambia su fondo.
    * **Fondo Activo:** `background: rgba(255, 255, 255, 0.05);` (en dark mode) o un gris muy tenue.
    * **Indicador Visual:** El item enfocado puede tener un borde izquierdo de `2px` usando `--accent-primary` para dejar clarísimo dónde está el cursor, incluso de reojo.

# 5. Interacción del Diagrama y Auto-Panning

Este módulo define cómo reacciona el *canvas* de Mermaid a medida que el usuario avanza por los pasos del tour. El objetivo es lograr un contraste dramático entre el "sujeto activo" y el "contexto", y garantizar que la cámara siga la acción de forma fluida.

## 5.1. Sistema de Foco Visual (The "Bold" Highlight)

Reemplazamos los resplandores borrosos (*glows*) por bordes sólidos y definidos que comunican precisión técnica.

* **Nodo Activo (Active Node):**
    * **Borde:** Sobrescribir el estilo de Mermaid para el nodo en foco. Usar un `stroke-width` de `3px` o `4px` con el color `--accent-primary` (ej. azul eléctrico).
    * **Fondo:** Mantener el fondo del nodo en un nivel de contraste legible (usando `--bg-surface`), pero el texto dentro del nodo activo debe brillar (usar `--text-primary` al 100% de opacidad).
    * **Elevación CSS:** Aplicar un sutil `transform: scale(1.02);` al nodo activo con una transición rápida (`transition: all 0.2s ease`).

* **Conectores Activos (Active Edges):**
    * Si el paso actual describe el flujo entre el Nodo A y el Nodo B, la flecha/línea conectora también debe iluminarse.
    * **Estilo:** `stroke-width: 2.5px; stroke: var(--accent-primary);`
    * **Animación (Opcional pero recomendado):** Si es técnicamente viable con las clases SVG de Mermaid, añadir una animación CSS sutil al `stroke-dashoffset` para que la línea parezca "fluir" hacia su destino.

## 5.2. Atenuación del Contexto (Dimming)

Para que el foco funcione, todo lo demás debe pasar a un segundo plano.

* **Nodos y Conectores Inactivos:**
    * **Opacidad:** Reducir la opacidad general de los elementos no involucrados en el paso actual a un `25%` o `30%` (`opacity: 0.3;`).
    * **Desaturación:** Aplicar un filtro de escala de grises suave (`filter: grayscale(80%);`) para que no compitan visualmente con el color de acento del nodo activo.
    * **Transición:** La atenuación debe ser suave (`transition: opacity 0.4s ease, filter 0.4s ease`).

## 5.3. Control de Cámara y Auto-Panning

El diagrama no puede quedar estático si el flujo lleva al usuario fuera de la pantalla. El *viewport* debe ajustarse automáticamente.

* **Centrado Inteligente (Smart Panning):**
    * Al cambiar de paso, calcular la posición (Bounding Box) del nodo activo (o grupo de nodos activos).
    * Aplicar una transformación (`transform: translate`) al contenedor del diagrama para mover el nodo activo hacia el centro de la pantalla.
* **Compensación de la UI (Safe Areas):**
    * **Crucial:** El nodo no debe centrarse en el medio exacto de la pantalla (50% / 50%), ya que el panel inferior ("Teleprompter") ocupa espacio.
    * El centro óptico de la cámara debe estar en el **tercio superior** de la pantalla (`Y: 35%` o `40%`) para dejar que el panel de texto viva abajo sin tapar el diagrama.
* **Curva de Animación:**
    * El movimiento de cámara debe sentirse orgánico, no mecánico. Usar una función de aceleración como `transition: transform 0.6s cubic-bezier(0.25, 1, 0.5, 1);` (Ease-out rápido al principio, desaceleración suave al final).

## 5.4. Sobrescritura de Temas de Mermaid (Mermaid Config)

Para que los colores de Mermaid coincidan perfectamente con nuestros Design Tokens, debemos inyectar las variables de CSS en la configuración de inicialización de Mermaid.

* Configurar `mermaid.initialize({ theme: 'base', themeVariables: { ... } })` mapeando los colores primarios y secundarios de los nodos a nuestras variables `--bg-surface`, `--text-primary`, `--border-subtle`, etc.
* Eliminar cualquier estilo en línea por defecto que Mermaid intente forzar que contradiga el *Dark Theme*.
# 6. Sistema de Alertas y Panel de Issues

Este módulo define cómo se presentan los errores de validación (ej. nodos Mermaid no encontrados, sintaxis YAML inválida). El objetivo es pasar de un volcado de texto crudo a una interfaz estructurada, escaneable y que invite a la resolución del problema, al estilo de los diagnósticos en VS Code o Vercel.

## 6.1. El Popover de Issues (Contenedor)

Al hacer clic en el botón `[! Issues]` del Header, se desplegará un panel flotante anclado a ese botón.

* **Posicionamiento:** Anclado debajo y alineado a la derecha del botón (`position: absolute; top: calc(100% + 8px); right: 0;`).
* **Estilo del Contenedor:**
    * Fondo: `--bg-surface` (sin transparencias para máxima legibilidad).
    * Borde: `1px solid var(--border-subtle)`.
    * Sombra: `box-shadow: 0 12px 24px -6px rgba(0, 0, 0, 0.5);`
    * Radio: `border-radius: var(--radius-md);` (ej. 8px o 12px).
    * Ancho: Fijo o máximo de `420px` para acomodar rutas de archivos largas sin romper el layout.
* **Cabecera del Popover (Header interno):**
    * Un área superior con `padding: 12px 16px; border-bottom: 1px solid var(--border-subtle);`.
    * Título: "Skipped Tours" o "Issues Detected" en `--font-sans`, `font-weight: 600`, color `--text-primary`.
    * Contador: Un *badge* pequeño al lado del título indicando la cantidad (ej. `2`).

## 6.2. Estructura del Ítem de Error (Issue Card)

Cada error individual en la lista no debe ser un simple párrafo de texto. Debe dividirse en jerarquías visuales.

* **Layout del Ítem:** `display: flex; flex-direction: column; gap: 4px; padding: 12px 16px; border-bottom: 1px solid var(--border-subtle);` (el último ítem no lleva borde inferior).
* **Fila 1: Archivo afectado (Contexto)**
    * Usar `--font-mono` en tamaño `0.75rem`.
    * Color: `--text-secondary`.
    * Mostrar la ruta truncada inteligentemente si es muy larga (ej. `.../fixtures/invalid-tour.yaml`).
* **Fila 2: El Mensaje de Error (El Problema)**
    * Usar `--font-sans` en tamaño `0.875rem` (`14px`).
    * Color: `--text-primary`.
    * Texto descriptivo claro, ej: "Step 1 focus references unknown Mermaid node id".
* **Fila 3: El Fragmento / Código (El Detalle Técnico)**
    * El valor exacto que falló (ej. `"missing_node"`) debe envolverse en un bloque de código en línea (`<code>`).
    * Fondo del tag code: `rgba(244, 63, 94, 0.15)` (un rojo semitransparente).
    * Color del texto del code: `--text-error` (ver sección 6.3).
    * Tipografía: `--font-mono`, tamaño `0.75rem`.

## 6.3. Paleta Semántica de Estados (Design Tokens)

Debemos agregar colores específicos para los estados de error y advertencia al sistema de Theming.

**Dark Theme (Alertas):**
* **Error / Crítico:**
    * `--text-error`: `#FDA4AF` (Rose 300).
    * `--border-error`: `rgba(244, 63, 94, 0.4)`.
    * `--bg-error-muted`: `rgba(244, 63, 94, 0.1)`.
* **Warning / Advertencia (Opcional):**
    * `--text-warning`: `#FCD34D` (Amber 300).
    * `--bg-warning-muted`: `rgba(245, 158, 11, 0.1)`.

## 6.4. Estado Vacío (Zero State) y Toasts

* **Zero State:** Si el usuario hace clic en "Issues" y no hay errores (o si los resuelve todos), el popover debe mostrar un mensaje amable y centrado. Ej. Ícono de *check* verde sutil, texto: "All clear. No issues found in current workspace."
* **Toasts (Notificaciones efímeras):** Para acciones exitosas (ej. "Ruta de archivo copiada al portapapeles"), usar una notificación flotante pequeña en la parte inferior derecha, con fondo `--bg-surface`, que desaparezca a los 3 segundos.
